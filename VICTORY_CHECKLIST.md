## ❓ Q&A Checklist

### During Questions

- [ ] Listen completely before answering
- [ ] Pause 1 second to gather thoughts
- [ ] Answer honestly ("Great question...")
- [ ] Reference specific features when relevant
- [ ] If unsure: "Our thinking is..." not "I don't know"
- [ ] Keep answers under 45 seconds
- [ ] Maintain eye contact
- [ ] Show enthusiasm

### Common Questions to Prepare

- [ ] "How do you get GPS data?" → Ready to answer
- [ ] "What about privacy?" → Ready to answer
- [ ] "How does this scale?" → Ready to answer
- [ ] "Business model?" → Ready to answer
- [ ] "Better than Google Maps?" → Ready to answer
- [ ] "What about fake GPS?" → Ready to answer
- [ ] "Technical stack?" → Ready to answer
- [ ] "How long to build?" → Ready to answer

### Red Flag Questions

If judge asks about:

- [ ] **Security:** Be honest about demo vs production
- [ ] **Scale:** Reference configurable architecture
- [ ] **Competition:** Emphasize unique LGU angle
- [ ] **Implementation:** Focus on value, not just tech

## 🎯 Success Indicators During Demo

### Good Signs

✅ Judges lean forward
✅ Judges nod during explanation
✅ Judges smile or look impressed
✅ Questions about deployment (not basics)
✅ Questions about business model
✅ "Interesting approach" comments
✅ Multiple questions asked
✅ Judges take notes
✅ Audience engagement

### Neutral Signs

➖ Few questions (could be clear or confusing)
➖ Technical questions (could be interest or confusion)
➖ Judges typing (taking notes or distracted)

### Warning Signs (Adapt Mid-Presentation)

⚠️ Judges look confused → Slow down, clarify
⚠️ Judges look bored → Add energy, emphasize impact
⚠️ Clock running out → Skip to most impressive features

## 🚨 Emergency Protocols

### If Technical Issues Occur

#### Demo won't load

- [ ] Use backup deployed URL
- [ ] Use backup laptop
- [ ] Use screenshot walkthrough
- [ ] Use demo video (if created)
- [ ] Explain verbally with gestures

#### Internet fails

- [ ] Local demo should work offline (maps need internet though)
- [ ] Switch to slides with screenshots
- [ ] Walk through features with images
- [ ] Emphasize architecture and thinking

#### Projector issues

- [ ] Continue on laptop screen
- [ ] Invite judges to see laptop up close
- [ ] Use slides as primary
- [ ] Don't panic, stay professional

### If Presentation Issues Occur

#### Forget what to say

- [ ] Take 2-second pause
- [ ] Look at [QUICK_REFERENCE.md](./QUICK_REFERENCE.md)
- [ ] Jump to next point you remember
- [ ] Don't apologize, keep going

#### Run out of time

- [ ] Jump to impact statement
- [ ] Close confidently
- [ ] Don't rush through everything

#### Tough question you can't answer

- [ ] "That's a great question"
- [ ] "Our thinking is [educated guess]"
- [ ] "We'd research that further in production"
- [ ] Never say "I don't know" - say "Here's our approach"
