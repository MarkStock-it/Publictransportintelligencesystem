## 🚀 How to Use This

### Before the Hackathon (Prep)

1. **Install & Test**
   ```bash
   npm install
   npm run dev
   ```
   - Verify everything works
   - Test all 4 views
   - Check mobile responsiveness

2. **Study the Docs**
   - Read `QUICKSTART.md` first
   - Review `DEMO_SCRIPT.md` thoroughly
   - Understand `TECHNICAL.md` for Q&A
   - Know `TROUBLESHOOTING.md` exists

3. **Practice Your Demo**
   - Follow the 3-minute script
   - Time yourself (aim for 2:45)
   - Practice smooth navigation
   - Prepare for Q&A using common questions

4. **Create Pitch Deck**
   - Use `PITCH_DECK.md` as outline
   - Design slides (PowerPoint/Google Slides)
   - Include screenshots of your app
   - Practice presentation 3-5 times

5. **Prepare Backup**
   - Deploy to Vercel/Netlify
   - Have demo URL as backup
   - PDF of slides on USB
   - Screenshot of working demo

### During the Hackathon

#### Setup Phase (First 30 mins)

1. Get to know teammates
2. Assign roles:
   - **Developer:** Handles code, customizations
   - **Designer:** Creates pitch deck, visuals
   - **Researcher:** Gathers data, impact metrics
   - **Presenter:** Masters demo script

3. Customize for your context:
   - Adjust routes if needed (in `jeepney-data.ts`)
   - Update team names in README
   - Add local context to pitch

#### Mid-Hackathon (Testing)

1. Test on multiple devices
2. Get feedback from mentors
3. Refine pitch based on feedback
4. Practice demo transitions
5. Prepare answers to specific judge questions

#### Pre-Demo (1 hour before)

1. **Technical Check**
   - Fresh `npm install`
   - Test live demo
   - Clear browser cache
   - Verify no console errors
   - Have backup URL ready

2. **Presentation Check**
   - Slides loaded and tested
   - Presenter dressed appropriately
   - Water bottle ready
   - Team roles clear

3. **Mental Prep**
   - Review key talking points
   - Deep breaths
   - Team huddle - "We got this!"

#### Demo Time (3-5 minutes)

**Follow this exact flow:**

1. **Opening (15s)**
   - Team intro
   - "We're presenting JeepWise..."

2. **Problem (30s)**
   - "Filipino commuters face uncertainty..."
   - Emotional connection

3. **Solution Demo (90s)**
   - Live Map (30s) - Show movement, click stop, click jeep
   - Route Intelligence (30s) - Show analytics
   - LGU Dashboard (30s) - Show charts and insights

4. **Impact (20s)**
   - "With JeepWise, commuters get visibility..."
   - Quantified metrics

5. **Close (15s)**
   - "This isn't just a tracker, it's an ecosystem"
   - Thank you

#### Q&A (Variable)

**Common Questions:**

**Q: "How do you get GPS data?"**
A: "For this demo, we're simulating. Production version uses driver mobile app with GPS pings every 5 seconds. We architected it for real-world deployment."

**Q: "What about privacy?"**
A: "We only track jeep locations, not personal data. Drivers identified by jeep number only. LGUs see aggregated analytics."

**Q: "Why is this better than Google Maps?"**
A: "Google Maps does navigation. JeepWise does public transport intelligence - arrivals, occupancy, and government planning tools. Different problems."

**Q: "Can this scale?"**
A: "Yes. Our route system is configurable. We can add new cities by simply providing GPS coordinates. Backend architecture ready for production."

**Q: "Business model?"**
A: "Three streams: LGU subscriptions for dashboard access, driver cooperative fees, and future API licensing. But social impact is our primary goal."

**Be confident. Be honest. Show passion.**
